Documentation - Data Model

1. Channel:

Channel is the method chosen by the customer to interact with the touchpoint, such as a “company’s website”, “mobile application”, or “call center”...

  • ID : ID of channel
  • Name : name of channel (website, app, store,...)

2. Traffic_Source:

This section contains information about the Traffic Source from which the session originated. For example, consider a man who watched an advertisement from Youtube about a book. He clicked on the advertisement link to access the home page and view the product details. Let ViewProductDetails be a touchpoint. The trafficSource of this touchpoint is defined as Youtube.

  • ID : ID of traffic source
  • Name : name of traffic source (Youtube, Google, Facebook,...)

3. Device_Browser:

This section contains information about the browser that customers used to interact with the company

  • ID : ID of browser
  • Name : name of browser (chrome, safari, edge,...)

4. Device_OS:

This section contains information about the operating system that customers used to interact with the company

  • ID : ID of operating system
  • Name : name of operating system (Windows, MacOS, Ubuntu,...)

5. Device_Category:

This section contains information about the customer’s device

  • ID : ID of device category
  • Name : name of device category (smartphone, tablet, laptop,...)

6. Interact_Item:

This section contains information about the item that customer interacted through touchpoint

  • ID : ID of interact item
  • Name : name of interact item (product, transaction, video, ...)

7. Experience_Emotion:

Using only one continuum of emotions–such as unhappy to happy– may fail to depict a customer’s experience

  • ID : ID of experience emotion
  • Name : name of experience emotion (happy, angry, satisfied, ...)

8. Action:

Action is a verb that’s denoted for each touchpoint. In other words, a touchpoint is described by an action. A customer can iterate a few times over the same action. Each action will be related to a service, a product, or a context it affects. In general, we call these services or products that belong to a company “interact item”

  • ID : ID of action
  • Name : name of action (ask for information, technical support, view product info, view product list, ...)

1. Touchpoint:

An interaction between customers and a company’s products or services at any time through a channel. A touchpoint was described by action through a channel. The stage indicates which stage this touchpoint belongs to. A touchpoint must provide

  • Customer_ID : ID of customer who executed the touchpoint
  • Action_Type : referenced to Action Model
  • Time : moment when customer executed the touchpoint
  • Channel_Type : referenced to Channel Model
  • Device_Browser : referenced to Device_Browser Model
  • Device_OS : referenced to Devices_OS Model
  • Device_Category : referenced to Devices_Category Model
  • Geo_Continent : geo-network data about continent tracked from device when customer executed the touchpoint
  • Geo_Country : geo-network data about country tracked from device when customer executed the touchpoint
  • Geo_City : geo-network data about city tracked from device when customer executed the touchpoint
  • Traffic_Source_Name : name of source where the current touchpoint was originated (ad, video, ...)
  • Traffic_Source_Url : URL of source where the current touchpoint was originated (ad, video, ...)
  • Store_ID : physical store ID where the customer directly interacts with the service of the company
  • Employee_ID : employee ID where the customer directly interacts with the service of the company
  • Webpage_Hostname : hostname of website which customer executed the touchpoint on
  • Webpage_Url : URL of website which customer executed the touchpoint on
  • Webpage_Title : title of page on website which customer executed the touchpoint on
  • App_Name : mobile app name that customer executed the touchpoint
  • App_Screen_Title : title of screen on mobile app that customer executed the touchpoint
  • Interact_Item_Type : referenced to Interact_Item Model
  • Interact_Item_ID : ID of item that customer interacted through the touchpoint (product_id, ...)
  • Interact_Item_Url : URL of item that customer interacted through the touchpoint (video_url, ...)
  • Interact_Item_Content : content of item that customer interacted through the touchpoint (email_content, ...)
  • Experience_Score : score that customer evaluated for the touchpoint experience
  • Experience_Emotion : referenced to Experience_Emotion Model
